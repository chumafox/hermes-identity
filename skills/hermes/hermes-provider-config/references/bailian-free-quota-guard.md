# Bailian Free Quota Guard (用完即停)

Управление защитой от платного режима в Alibaba Cloud Bailian (百炼).

## Где находится

Консоль → Модели → Модель用量 → вкладка «免费额度».

URL: `https://bailian.console.aliyun.com/cn-beijing?tab=model#/model-usage/free-quota`

## Что делают кнопки

Фиолетовая кнопка **«批量操作免费额度用完即停»** → дропдаун:

| Кнопка | Действие |
|--------|----------|
| **批量开启** | Разрешить платный режим после исчерпания квоты (выключить защиту) |
| **批量关闭** | Включить «用完即停» — только free quota, после исчерпания → 403 |

⚠️ **Путаница:** «批量**关闭**» = включить защиту. Логика: закрыть платный режим = оставить только бесплатный.

## Как работает защита

Когда для модели включено «用完即停»:
- API потребляет ТОЛЬКО бесплатную квоту
- При исчерпании → ошибка 403 `AllocationQuota.FreeTierOnly.`
- **Никаких платных списаний, никакого ухода в минус**

## Статусы моделей

В колонке «状态» три возможных значения:

| Статус | Значение |
|--------|----------|
| **未开启** | Защита НЕ включена (но поддерживается) |
| **已开启** | Защита включена |
| **不支持开启** | Защита НЕ поддерживается для этой модели |

Фильтр в заголовке колонки «状态» позволяет быстро найти все модели с определённым статусом: нажать filter → выбрать статус → «确定». Для аудита: фильтр «已开启» → смотрим pagination (сколько моделей защищено), фильтр «不支持开启» → смотрим какие модели беззащитны.

### «不支持开启» — что это значит

Модели со статусом «不支持开启» не имеют ссылки «免费额度用完即停» в колонке «操作» — только «购买节省计划». Это **платные или сторонние** модели:

- **Qwen платные:** qwen3.7-max, qwen3.6-flash-2026-04-16, qwen-deep-research
- **Сторонние:** kimi/kimi-k2.5, kimi/kimi-k2.6
- **Сторонние:** MiniMax/MiniMax-M2.5, MiniMax/MiniMax-M2.7, MiniMax/MiniMax-M3
- **Сторонние:** deepseek-v4-flash, deepseek-v4-pro

⚠️ **qwen3.7-max** — особый случай: бесплатная квота ЕСТЬ (1M токенов), но «用完即停» не поддерживается. Статус «不支持开启» ≠ «нет квоты».

## ⚠️ Необратимость

> «一旦功能开启，若您需要关闭本功能，需要在免费额度完全消耗后再进行关闭»

**Включил → выключить НЕЛЬЗЯ, пока квота не исчерпана полностью.**

Если передумал, а квота ещё есть — кнопка выключения недоступна.

## Как включить

**Через индивидуальные кнопки (надёжный способ):**
У каждой модели в колонке «操作» есть ссылка **«免费额度用完即停»** — нажать для включения защиты на конкретной модели.

**Через batch-операцию (ненадёжно):**
Кнопка «批量操作免费额度用完即停» → «批量关闭». 

⚠️ **Известная проблема (Jun 2026):** в таблице может не быть чекбоксов для выбора моделей. Batch-операция без выбранных моделей не срабатывает — страница перезагружается, но статусы остаются «未开启». Проверять наличие чекбоксов перед попыткой batch-операции.

После этого в столбце «状态» появится «已开启» вместо «未开启».

## Проверка статуса через CDP

Как найти PAGE_TARGET_ID — см. `browser-content-extraction` skill, секция «Multi-Tab CDP Targeting».

### Быстрый аудит через фильтр (рекомендуется)

Вместо пагинации через все страницы — использовать фильтр по статусу:

```javascript
// 1. Кликнуть filter в заголовке «状态»
document.querySelector('.efm_ant-table-filter-trigger').click()

// 2. Выбрать статус
Array.from(document.querySelectorAll('.efm_ant-table-filter-dropdown:not(.efm_ant-dropdown-hidden) .efm_ant-dropdown-menu-item'))
  .find(i => i.textContent.trim() === '已开启')?.click()

// 3. Подтвердить
Array.from(document.querySelectorAll('.efm_ant-table-filter-dropdown:not(.efm_ant-dropdown-hidden) button'))
  .find(b => b.textContent.trim() === '确定')?.click()

// 4. Прочитать pagination — там будет «共N条» = количество моделей с этим статусом
document.querySelector('[class*=pagination]')?.textContent?.match(/共(\d+)条/)?.[1]
```

### Построчное чтение таблицы

```javascript
// Все модели с защитой
Array.from(document.querySelectorAll('table tbody tr'))
  .filter(r => r.textContent.includes('已开启'))
  .map(r => r.querySelector('td:first-child')?.textContent?.trim())
  .filter(Boolean)

// Все статусы
Array.from(document.querySelectorAll('table tbody tr td:nth-child(4)'))
  .map(td => td.textContent.trim())
  .filter(s => s !== '')

// Статистика по странице
JSON.stringify({
  total: document.querySelectorAll('table tbody tr').length,
  statuses: Array.from(document.querySelectorAll('table tbody tr td:nth-child(4)'))
    .map(td => td.textContent.trim())
    .reduce((acc,s) => {acc[s]=(acc[s]||0)+1;return acc},{})
})
```

## Модели без free quota (36 штук)

Показываются отдельно в блоке «无免费额度模型» и в таблице со статусом «不支持开启». Для них «用完即停» не имеет смысла — они всегда платные.

Важные платные модели:
- `deepseek-v4-flash`, `deepseek-v4-pro` (через Bailian — post-paid)
- `kimi-k2.5`, `kimi-k2.6`
- `MiniMax-*` (все)
- `qwen3.7-max` (⚠️ квота есть, но «用完即停» не поддерживается)
- `qwen3.6-flash-2026-04-16`
- `qwen-deep-research`

## Как отличить платную модель от бесплатной

- Префиксы `vanchin/`, `siliconflow/`, `kimi/`, `MiniMax/`, `ZHIPU/` → почти всегда платные
- Модели без префикса (`qwen-*`, `glm-*`) → почти всегда бесплатные
- Точный статус: смотреть в колонке «免费额度剩余量» — если пусто, модель платная
