# Ghostty: Russian Layout Keybindings

## Проблема

Бинды в Ghostty (`keybind = ctrl+]=next_tab`) работают только в английской раскладке. В русской ЙЦУКЕН не срабатывают.

## Причина

Ghostty матчит клавиши по **Unicode codepoint** (символу), который генерирует нажатие, а не по физической позиции клавиши. В русской раскладке:

- Физическая клавиша `]` генерирует `ъ`
- Физическая клавиша `[` генерирует `х`

Поскольку бинд зарегистрирован на символ `]`, а нажатие в русской раскладке генерирует `ъ` — Ghostty не находит совпадения.

## Предпочтительное решение: Named keys

Если задача решается named keys (Tab, PageUp, F-клавиши, Enter и т.д.) — не дублировать. Named keys матчатся по USB HID коду физической клавиши и работают во всех раскладках без исключения.

```
keybind = ctrl+tab=next_tab
keybind = ctrl+shift+tab=previous_tab
```

Актуально для: переключение табов, сплитов, скроллинг, копирование/вставка.

## Fallback: дублирование для символьных клавиш

Когда нужная клавиша не имеет named-имени (например `[`, `]`, `;`, `'`) — добавить дублирующий бинд на символ другой раскладки:

```
# English layout
keybind = super+shift+]=next_tab
keybind = super+shift+[=previous_tab
# Russian layout
keybind = super+shift+ъ=next_tab
keybind = super+shift+х=previous_tab
```

## Как это работает

Когда Ghostty получает нажатие, он проверяет какой символ (Unicode codepoint) произвела клавиша с учётом модификаторов. Если символ совпадает с любым зарегистрированным биндом — action выполняется.

В US раскладке:
- клавиша `]` (с Shift+Super) → символ `}` → бинд `super+shift+]` матчится по базовому символу `]`

В русской раскладке:
- та же физическая клавиша (с Shift+Super) → символ `Ъ` (или `ъ` без shift)
- бинд `super+shift+ъ` матчится: клавиша производит `ъ` как базовый символ, Shift как модификатор, Super как модификатор

## Проверка

Вывести список активных биндов:
```
ghostty +list-keybinds
```

Стандартные бинды:
```
ghostty +list-keybinds --default
```

## Синтаксис

**Правильно**: `keybind = trigger=action` (пробелы вокруг первого `=`, нет пробелов вокруг второго)

**Ошибка**: `keybind = trigger = action` — Ghostty выдаст `error.InvalidAction`

## Маппинг физических клавиш US → Russian (ЙЦУКЕН)

| Физическая клавиша | US (без Shift) | Russian (без Shift) |
|---|---|---|
| Key right of P | `[` | `х` |
| Key right of [` | `]` | `ъ` |
| Key right of L | `;` | `ж` |
| Key right of ; | `'` | `э` |
| Key right of M | `,` | `б` |
| Key right of , | `.` | `ю` |
