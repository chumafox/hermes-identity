# WiFi Speed Test: Headless Mac

Измерение пропускной способности WiFi между экранным Mac (dispo) и безголовым Mac (admin) через iperf3.

## Когда использовать

- Пользователь просит замерить скорость между Mac в рамках корабельной WiFi сети
- Нужно проверить качество WiFi соединения между Mac
- Диагностика сетевых проблем между Mac

## Параметры сети

### WiFi корабля (192.168.102.0/23)

| Параметр | Значение |
|----------|----------|
| Безголовый Mac (WiFi, статика) | 192.168.103.70 |
| Экранный Mac (WiFi, DHCP) | 192.168.103.192 |

### ZTE LAN (в каюте, 192.168.0.0/24)

| Параметр | Значение |
|----------|----------|
| Безголовый Mac (ZTE USB, DHCP) | 192.168.0.96 |
| Экранный Mac (ZTE WiFi, DHCP) | 192.168.0.82 |
| Шлюз | 192.168.0.1 |

### Общее

| Параметр | Значение |
|----------|----------|
| SSH ключ | ~/.ssh/id_ed25519_hermes |
| Пользователь безголового | admin |
| MAC адрес безголового (WiFi) | bc:d0:74:46:20:0e |

## Инструкция

### 1. Убедиться, что iperf3 есть на безголовом Mac

```bash
ssh -i ~/.ssh/id_ed25519_hermes admin@<IP_безголового> "iperf3 --version 2>&1 || echo 'not installed'"
```

IP безголового: `192.168.103.70` (через WiFi корабля) или `192.168.0.96` (через ZTE LAN).

Если не установлен — скопировать с экранного Mac:

```bash
scp -i ~/.ssh/id_ed25519_hermes /opt/homebrew/bin/iperf3 admin@192.168.103.70:/tmp/iperf3
ssh -i ~/.ssh/id_ed25519_hermes admin@192.168.103.70 "chmod +x /tmp/iperf3"
```

### 2. Запустить iperf3 сервер на безголовом Mac

```bash
ssh -i ~/.ssh/id_ed25519_hermes admin@<IP_безголового> \
  "nohup /tmp/iperf3 -s > /tmp/iperf3-server.log 2>&1 & sleep 0.5 && echo server_started"
```

### 3. Замерить скорость (1 поток)

```bash
iperf3 -c <IP_безголового> -t 10 -P 1 -f m
```

### 4. Замерить скорость (4 потока)

```bash
iperf3 -c <IP_безголового> -t 10 -P 4 -f m
```

### 5. Остановить сервер

```bash
ssh -i ~/.ssh/id_ed25519_hermes admin@<IP_безголового> "pkill iperf3"
```

Где `<IP_безголового>`:
- `192.168.103.70` — через WiFi корабля (для теста между палубами)
- `192.168.0.96` — через ZTE LAN (для теста в каюте)

Оба Mac в одной точке (каюта), соединение через ZTE роутер. Ожидаемая скорость — 100-400 Mbit/s (зависит от ZTE, 802.11ac/ax).

## Реальные замеры (корабль, 802.11ax 5 ГГц)

| Сценарий | 1 поток | 4 потока | Пинг | Retr |
|----------|---------|----------|------|------|
| Одна каюта (ранний тест) | ~120 Mbit/s | ~163 Mbit/s | ~14 ms | 380 / 2106 |
| Одна каюта (поздний тест, др. время) | ~155 Mbit/s | ~157 Mbit/s | 93-240 ms | 895 / 1949 |
| Разные палубы | ~57 Mbit/s | ~86 Mbit/s | 11-85 ms | 3 / 0 |

Вывод: многопоточный тест стабильнее на расстоянии (меньше CWND collapse).  
Screen Sharing комфортно работает даже на разных палубах (57-86 Mbit/s).  
Скорость в каюте сильно зависит от времени суток и загрузки WiFi сети корабля (120-155 Mbit/s).

|------|----------|
| Download 10 MB | ~34 Mbit/s (2.3 sec) |

## Примечания

- iperf3 скопирован в /tmp — не сохраняется после перезагрузки безголового Mac. Если Mac перезагружался, скопировать заново (шаг 1).
- Безголовый Mac имеет два IP: 192.168.103.70 (WiFi корабля, статика) и 192.168.0.96 (ZTE LAN, DHCP). Для теста используй нужный IP в зависимости от того, через какую сеть проверяешь.
- Если IP безголового изменился — найти через mDNS: `ping -c 1 admin.local && arp -a | grep bc:d0:74:46:20:0e`
- Тест использует TCP; для теста UDP добавить флаг `-u` к обеим командам
- Реальные замеры с этого корабля — в `references/benchmarks.md`
- Для проверки интернета через SOCKS5 туннель не используй `ifconfig.me` — он падает с exit 52. Используй `http://httpbin.org/ip`.
- Google заблокирован China Mobile — не проверять интернет через Google. Использовать Baidu, httpbin или ipify.
