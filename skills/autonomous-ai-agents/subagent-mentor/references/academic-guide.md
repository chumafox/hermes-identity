# **Архитектура и оркестрация саб-агентов в интерфейсах командной строки: Исчерпывающее руководство по настройке, изоляции контекста и мониторингу**

Глубокая трансформация инструментов разработки на базе искусственного интеллекта, произошедшая к 2026 году, ознаменовала переход от парадигмы единого, монолитного агента-помощника к сложным, распределенным многоагентным экосистемам, функционирующим непосредственно в интерфейсе командной строки (CLI).

## Фундаментальные принципы изоляции, делегирования и терминального рендеринга

Механизмы изоляции контекста проектируются с целью минимизации информационной избыточности. Саб-агенты запускаются с жестко ограниченным или абсолютно чистым контекстом. В отличие от монолитной сессии, в которой ИИ-модель обладает полным доступом к многостраничной истории переписки, новый инстанс саб-агента получает только системный промпт (задающий его роль и компетенции), список разрешенных инструментов вызова (tool calls) и конкретную полезную нагрузку задачи.

Модели исполнения саб-агентов классифицируются на две основные категории: блокирующие (Foreground) и фоновые (Background) процессы. В блокирующем режиме родительский процесс-оркестратор приостанавливает выполнение собственного цикла и ожидает сигнала о завершении работы саб-агента. Фоновый режим предполагает делегирование задачи в независимый параллельный процесс.

Отслеживание десятков параллельных асинхронных процессов в моноширинном текстовом интерфейсе решается через TUI-оверлеи — динамические текстовые панели, вызываемые через /agents или /tasks.

## Antigravity CLI (agy)

Преемник Gemini CLI, написан на Go. Функционал делегирования обладает высокой степенью автономности. Оркестратор системы способен самостоятельно профилировать пользовательский запрос, производить декомпозицию задачи и динамически порождать специализированные роли.

Встроенные саб-агенты: research (навигация по codebase), browser (headless Chrome через MCP), self (клон оркестратора).

Создание кастомных саб-агентов через define_subagent — коммерческая функция Ultra Plan. Без подписки — только эфемерные агенты.

Мониторинг: /agents (Agent Manager Panel), Enter — Subagent Detail View, Esc — возврат. /tasks для bash-скриптов.

## OpenCode

Высокомодульный терминальный инструмент на Go. Агенты классифицируются на primary (Build, Plan) и subagent (general, explore, scout).

Конфигурация: opencode.json или Markdown + YAML frontmatter в ~/.config/opencode/agents/ или .opencode/agents/.

Параметры: description (критически важен), mode, model, temperature, steps.

Permissions: allow/ask/deny для каждого инструмента, glob-паттерны (git status: allow, git push: ask), permission.task для запрета каскадного вызова.

Навигация: \<Leader\>+Down в сессию, Right/Left между саб-агентами, Up возврат.

## Hermes Agent

Агентная ОС от Nous Research. Основной инструмент — delegate_task(goal, context, toolsets, role).

Важно: саб-агенты стартуют tabula rasa — без доступа к истории родителя. Контекст передается явно.

Параллельное выполнение: batch mode (tasks=[]), max_concurrent_children (default 3). Глубина рекурсии: max_spawn_depth. Orchestrator role для создания под-агентов.

Мониторинг: /agents в TUI (дерево, токены, стоимость, file-touch summaries). Hermes Kanban: Triage -> Todo -> Ready -> In progress -> Blocked -> Done. Автоматическое восстановление при падении агента.

## Cursor CLI

Форк VS Code. Саб-агенты — независимые фоновые воркеры. Конфигурация в .cursor/agents/ или ~/.cursor/agents/.

Параметры: name, description, model (inherit/fast), readonly, is_background.

Изоляция через git-worktree-toolbox — каждый агент в своем worktree на своей ветке.

Делегирование через Task tool. Параллелизм — все вызовы в одном ответе. Мониторинг через Agents Window.

## Сравнительная таблица

| Характеристика | Antigravity CLI | OpenCode CLI | Hermes Agent | Cursor CLI |
|---|---|---|---|---|
| Конфигурация | Скрытая логика | JSON или Markdown+YAML | YAML, Kanban | Markdown+YAML |
| Мониторинг | /agents, /tasks | Вложенные сессии TUI | /agents в TUI, веб | Agents Window |
| Рекурсия | Да | permission.task | max_spawn_depth | Task tool |
| Конфликты FS | Git Worktrees | Логическая изоляция | Docker/Modal | git-worktree-toolbox |
| Лицензия | Ultra Plan | Open Source | Open Source | Pro |

## Протокол инициализации ИИ-инструктора

**Фаза 1: Идентификация.** Спросить: "В какой CLI-среде будем настраивать саб-агентов: Antigravity (agy), OpenCode, Hermes или Cursor?"

**Фаза 2: Пошаговое обучение (4 этапа):**
1. Конфигурация — где создавать файлы, формат, поля
2. Безопасность — permissions, принцип минимальных прав
3. Делегирование — автоматическое (по description) и явное (@agent)
4. Мониторинг — TUI-оверлеи, Todo-листы, дашборды

**Фаза 3: Контроль.** Требовать выполнить команду в терминале и описать результат. Не переходить к следующему шагу без подтверждения.
