# Local Web App (Personal Tool)

Build a single-user, local-only web application that runs on macOS via `npm run dev` or serves from `dist/`.

## Stack

- **Runtime:** Node.js via npm
- **Framework:** React 19+ with TypeScript
- **Build:** Vite 8+ (generated by `npm create vite@latest`)
- **CSS:** Tailwind 4 (`tailwindcss` + `@tailwindcss/vite` plugin)
- **Storage:** localStorage (serialised JSON, saved on every change via `useEffect`)
- **No backend, no API, no auth, no deployment** — purely local

## User Preferences (jenyanovak)

- **Language:** Russian interface (UI text, button labels, date formatting, errors)
- **Theme:** Dark only (`@theme` custom colours, never light)
- **All inputs are `select`/dropdowns** — never free-text fields for data entry. Use `<select>` for weight, reps, rest, exercise names.
- **Data model in `data.ts`** — all types, exercise lists, storage helpers, and query functions in one file
- **Build:** `npm run build` (tsc -b + vite build). Must compile cleanly.
- **Directory:** `~/Projects/active/<project-name>/`

### Data Layer Pattern (data.ts)

```ts
const STORAGE_KEY = 'app-name-data';

export function loadData<T>(): T[] {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    return raw ? JSON.parse(raw) : [];
  } catch { return []; }
}

export function saveData<T>(data: T[]): void {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(data));
}
```

### Known Pitfalls

- **rolldown native binding**: on macOS 26.5+ with npm, `node_modules` can get corrupted. Fix: `rm -rf node_modules package-lock.json && npm install`
- **Tailwind install timeouts in China**: use `--prefer-offline` or retry. The install is small (~21 packages), just needs patience.
- **noUnusedLocals/noUnusedParameters**: tsconfig.app.json enables these. Every import and prop must be used, or the build fails on `tsc -b`.
- **Strongly prefer full-file writes** (`write_file`) for .tsx files rather than patches — the linter doesn't validate partial context and patch can introduce import issues.
